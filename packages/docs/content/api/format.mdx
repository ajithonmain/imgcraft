# Format

<Badge variant="node">Node</Badge> <Badge variant="browser">Browser</Badge>

Convert images between formats with per-format quality and encoding options.

## Format methods

```typescript
.jpeg(options?: JpegOptions): Pipeline
.png(options?: PngOptions): Pipeline
.webp(options?: WebpOptions): Pipeline
.avif(options?: AvifOptions): Pipeline
```

Or use the generic method:

```typescript
.format(type: 'jpeg' | 'png' | 'webp' | 'avif', options?: FormatOptions): Pipeline
```

## JPEG options

| Prop | Type | Default | Description |
|---|---|---|---|
| quality | `number` | `80` | Quality 1–100. Lower = smaller file. |
| progressive | `boolean` | `false` | Enable progressive (interlaced) JPEG. |
| mozjpeg | `boolean` | `false` | Use MozJPEG encoder for better compression. |

## PNG options

| Prop | Type | Default | Description |
|---|---|---|---|
| quality | `number` | `100` | Quality 1–100 (lossless if 100). |
| compressionLevel | `number` | `6` | zlib compression level 0–9. |
| progressive | `boolean` | `false` | Enable interlaced PNG. |

## WebP options

| Prop | Type | Default | Description |
|---|---|---|---|
| quality | `number` | `80` | Quality 1–100. |
| lossless | `boolean` | `false` | Enable lossless WebP. |
| effort | `number` | `4` | Compression effort 0–6. Higher = smaller, slower. |

## AVIF options

| Prop | Type | Default | Description |
|---|---|---|---|
| quality | `number` | `50` | Quality 1–100. |
| effort | `number` | `4` | Compression effort 0–9. |
| lossless | `boolean` | `false` | Enable lossless encoding. |

## Examples

```typescript
// Convert to WebP at 85% quality
await img('photo.jpg').webp({ quality: 85 }).toBuffer()

// Lossless PNG
await img('photo.jpg').png({ quality: 100 }).toBuffer()

// High-efficiency AVIF
await img('photo.jpg').avif({ quality: 50, effort: 6 }).toBuffer()

// Generic format method
await img('photo.jpg').format('webp', { quality: 80 }).toBuffer()
```

<Callout type="info">
  WebP typically achieves 25–35% smaller files than JPEG at equivalent visual quality. AVIF goes further but is slower to encode.
</Callout>
